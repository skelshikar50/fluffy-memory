Market Notes

What top analysts do on GitHub & LinkedIn

Researched from 50+ real data analyst profiles in 2025 — what gets people hired vs. ignored.

🐙
GitHub Best Practices · 2025
Always include a README.md for every project — problem statement, tools, steps, and result. Repos without README get skipped by recruiters.
Use clean folder structure: /data, /notebooks, /scripts, /reports. Shows you think about organisation.
Pin your best 6 repos on your profile. Include a profile README.md introducing yourself with your skills and contact.
Show Jupyter notebooks for Python analysis — they combine code + explanation, exactly what hiring managers look for.
Add a live demo link or Power BI published report link in the repo description where possible.
💼
LinkedIn Best Practices · 2025
Post project walkthroughs as carousels (PDF slides) — they get 3× more reach than plain text posts in analytics.
Add projects under Featured section with a cover image, description, and link to GitHub or live report.
Write a strong About section (300–500 chars) stating your speciality, tools, and what kind of problems you solve.
Include specific outcomes in experience bullets — "Built Power BI dashboard that reduced reporting time by 40%" beats "created dashboards".
Use #dataanalytics #powerbi #sql #python in posts — LinkedIn's algorithm surfaces these to recruiters actively searching.
⭐
What Hiring Managers Look For · 2025
End-to-end projects beat isolated SQL queries — show you can take a problem from raw data to a business insight.
Projects using real or realistic datasets (Kaggle, government data, company-style data) score higher than toy examples.
Show Power BI + SQL together in one project — it proves you can do both the engineering and visualisation layer.
Include a business context in every project description: "Company had problem X, I solved it by doing Y, result was Z."
